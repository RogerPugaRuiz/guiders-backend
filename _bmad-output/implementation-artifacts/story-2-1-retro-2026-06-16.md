## 🔧 Lo que fue difícil

1. **Subagente `@tdd-generator` no retornó output** (2 invocaciones vacías)
   - Plan B: escribir tests manualmente con patrón de Story 1.3 (mocks `const`, buildApp antes de mockResolvedValue)
   - Score: 7.5/10 (similar a Story 1.3, no mejoró en iteración 2)

2. **E2E test pattern sigue siendo sutil**
   - Primer intento: 12 tests fallaban con `TypeError: Cannot read 'isErr' of undefined`
   - Causa: `mockReset` en `buildApp` borraba mocks configurados antes
   - Fix: reordenar `app = await buildApp(...)` ANTES de `mockX.fn().mockResolvedValue(...)`

3. **Test T1 (validation tests don't validate) — wake-up call**
   - 7 tests de validación usaban `instanceof BffSessionError` genérico
   - Pasarían aunque `validateSessionData` retornara `null` siempre
   - Fix: assertions específicas por branch + 2 tests nuevos (companyId > 256, embedTokenRef vacío)

4. **Cookie name collision (F1/E5) — deferred pero ticking time bomb**
   - Decisión arquitectónica (cookie name = `access_token`) colisionará con OIDC cuando 2.6 normalice
   - Resolverlo ahora rompería la estrategia para 2.6
   - Documentado en `bff/AGENTS.md#known-limitation`
